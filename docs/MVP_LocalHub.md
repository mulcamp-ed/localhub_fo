## 2. 문제 & 타깃 (Problem)

- 누구: 서울을 여행·생활하는 주민과 관광객
- 문제: 공공 관광데이터는 다루기 어렵고, 정보를 가볍게 나눌 익명 창구가 없다.
- 니즈: 로그인 없이 맛집·관광지·축제 정보를 검색하고 공유하고 싶다.

## 3. 솔루션 & 핵심 가치 (Solution)

정제된 공공데이터를 지도·대시보드·챗봇으로 쉽게 소비하고, 회원가입 없는 익명 게시판으로 공유 진입장벽을 없앤다. 별도 서버 없이 브라우저에서 즉시 동작(정적 SPA + localStorage).

> **핵심 가치**: "로그인 없이, 서울 지역 정보를 검색하고 나누는 가장 가벼운 방법"

## 4. 핵심 기능 (Must-have)

| # | 기능 | 설명 |
|---|------|------|
| 1 | 익명 커뮤니티 CRUD | 서울 게시판 목록·상세·작성·수정·삭제 (localStorage) |
| 2 | 비밀번호 권한 | 작성 시 수정용 비밀번호 저장, 수정·삭제 시 프론트 검증 |
| 3 | 지역정보 챗봇 | 제공 JSON 기반 자연어 Q&A (키 있으면 LLM, 없으면 로컬 검색) |
| 4 | 공공데이터 연동 | 서울 TourAPI 8종(8,150건) 프론트 직접 로드 |

## 5. 범위 밖 & 선택 기능 (Scope)

- 제외: 회원가입·로그인, 서버·DB, 기기 간 동기화, 비밀번호 암호화(교육용 평문)
- 선택(구현 완료): 지도(Leaflet), 대시보드(Chart.js), 축제 캘린더, 실시간 알림, 검색·태그·좋아요·북마크·공유

## 6. 기술 스택 & 데이터 (Tech & Data)

| 항목 | 내용 |
|------|------|
| Frontend | Vue.js 3 (Vite) + Vue Router + Pinia |
| 저장/AI/시각화 | localStorage · OpenAI 호환(GMS) · Leaflet, Chart.js |
| 배포 | Netlify (repo 연동) |
| 데이터 | 한국관광공사 TourAPI 4.0 — 서울 8종 8,150건 / 공공누리 제3유형(출처표시·변경금지) / 2026-06 수집 |

## 7. 성공 기준 (Definition of Done)

- 서울 게시판 CRUD 동작 및 올바른 비밀번호로만 수정·삭제
- 챗봇이 제공 데이터를 근거로 응답(키 없으면 로컬 검색 폴백)
- 지도·대시보드·캘린더가 실제 서울 데이터로 렌더링
- 빌드 성공 + Netlify 배포 URL 정상 동작, `.env` 미포함
