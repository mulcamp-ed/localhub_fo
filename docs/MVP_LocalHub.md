# MVP 정의서 — LocalHub

> `MVP_TEMPLATE.md` 양식에 LocalHub 공문(v8)을 기반으로 실제 MVP를 추출·작성한 문서입니다.

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | LocalHub — 지역 정보 공유 익명 커뮤니티 |
| 한 줄 소개 | 공공데이터(한국관광공사 TourAPI)를 활용해 지역 주민·관광객이 자유롭게 정보를 공유·소비하는 백엔드 없는 익명 커뮤니티 SPA |
| 작성일 / 버전 | 2026-07-02 / v1.0 |
| 개발 기간 | 2026-07-02 착수 ~ 2026-07-16(목) 15:00 납기 (본 WBS는 3일 집중 개발 기준) |

## 2. 문제 정의 (Problem)

- **누가**: 특정 권역의 지역 주민 및 여행 계획 중인 관광객
- **어떤 상황에서**: 낯선 지역의 맛집·관광지·축제 정보를 찾고, 실제 방문자의 생생한 후기를 나누고 싶을 때
- **어떤 불편/니즈가 있는가**:
  - 공공데이터는 흩어져 있고 원본 API는 일반 사용자가 직접 다루기 어렵다.
  - 지역 정보를 가볍게 남기고 싶은데 회원가입·로그인이 진입장벽이 된다.
  - 자연어로 "이 근처 맛집 알려줘" 처럼 물어볼 창구가 없다.

## 3. 솔루션 가설 (Solution)

정제된 공공데이터를 지도·대시보드·챗봇으로 **소비하기 쉽게** 시각화하고, **회원가입 없는 익명 게시판**으로 **공유의 진입장벽을 제거**한다. 별도 백엔드 없이 브라우저에서 즉시 동작(정적 SPA + localStorage)하여 배포·운영 비용을 최소화한다.

> **핵심 가치 제안**: "로그인 없이, 우리 동네 정보를 검색하고 나누는 가장 가벼운 방법"

## 4. 핵심 사용자 시나리오 (User Story)

- 관광객으로서 여행지를 정하기 위해 **지도에서 권역별 관광지·맛집을 한눈에 보고** 싶다.
- 주민으로서 정보를 나누기 위해 **회원가입 없이 익명으로 글을 쓰고, 내 비밀번호로만 수정·삭제**하고 싶다.
- 사용자로서 빠르게 정보를 얻기 위해 **챗봇에게 자연어로 "한강 근처 관광지 알려줘"라고 묻고** 싶다.
- 기획자로서 현황 파악을 위해 **대시보드에서 카테고리·권역·인기 지역 통계를 보고** 싶다.

## 5. MVP 기능 범위 (Scope)

### 5.1 반드시 포함 (Must have — MVP)

| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | 익명 커뮤니티 CRUD | 5개 권역 게시판 목록/상세/작성/수정/삭제, localStorage 저장 | P0 |
| 2 | 비밀번호 권한 검증 | 작성 시 수정용 비밀번호 저장, 수정·삭제 시 프론트 로직으로 일치 확인 | P0 |
| 3 | 지역정보 챗봇 | 제공 JSON 기반 자연어 Q&A, 플로팅 위젯, 대화 히스토리, 모바일 대응 | P0 |
| 4 | 공공데이터 연동 | 서울 TourAPI JSON 8종을 프론트에서 직접 로드·처리 | P0 |
| 5 | 반응형 SPA/배포 | Vue3 SPA, Netlify 배포 | P0 |

### 5.2 포함하지 않음 (Out of scope — 차기)

| 기능 | 제외 사유 |
|------|-----------|
| 회원가입·로그인·권한 체계 | 공문상 익명 커뮤니티로 명시 (인증 미적용) |
| 서버·DB·실시간 WebSocket | 백엔드 없는 정적 SPA 구조 (localStorage로 대체) |
| 게시글의 기기 간 동기화 | localStorage 특성상 작성 브라우저에만 저장됨 |
| 비밀번호 암호화 | 서버 부재로 평문 저장 — 교육 목적의 의도된 설계 |

### 5.3 선택 기능 (Nice to have — 구현 완료)

> 공문상 "최소 1개" 요구. 본 프로젝트는 아래 5종을 모두 구현.

| 기능 | 내용 |
|------|------|
| 지도 시각화 | Leaflet + OSM으로 관광지·맛집 등 POI 핀 시각화 및 카테고리 필터 |
| 데이터 대시보드 | Chart.js로 카테고리별 건수, 권역별 게시글, 자치구 TOP10 통계 |
| 축제 캘린더 | 축제·행사 데이터를 월별 캘린더로 시각화 (정보 갱신일 기준) |
| 실시간 알림 | 서버 없이 localStorage polling으로 새 게시글 토스트 알림 |
| 커뮤니티 부가기능 | 조회수·검색·태그·북마크·좋아요·링크 공유(OG 태그) |

## 6. 기술 스택 (Tech Stack)

| 구분 | 기술 | 선택 이유 |
|------|------|-----------|
| Frontend | Vue.js 3 (Vite) + Vue Router + Pinia | 공문 지정, 빠른 HMR·경량 SPA |
| Backend | 없음 (Serverless / Static) | 공문상 별도 백엔드 서버 없음 |
| Database | 브라우저 localStorage | 게시글 영속화 (서버 부재) |
| 시각화 | Leaflet, Chart.js | 지도/차트 표준 오픈소스 라이브러리 |
| AI | OpenAI 호환 게이트웨이(GMS) 직접 호출 | 자연어 질의응답, `VITE_` 환경변수 키 관리 |
| 배포 | Netlify | 공문 지정, repo 연동 정적 배포 |

## 7. 데이터 명세 (Data)

### 7.1 활용 데이터 목록

| 데이터명 | 출처 | 라이선스 | 수집일 | 활용 방향 |
|----------|------|----------|--------|-----------|
| 국문 관광정보 서비스 (TourAPI 4.0) — 서울 8종(관광지/음식점/축제/문화시설/레포츠/숙박/쇼핑/여행코스, 총 8,150건) | 한국관광공사 (data.go.kr/data/15101578) | 공공누리 제3유형 (출처표시·변경금지) | 2026-06 | 지도 핀, 대시보드 통계, 축제 캘린더, 챗봇 지식베이스 |

> **출처 표기**: "이 서비스는 한국관광공사 Tour API(TourAPI 4.0)의 데이터를 활용하였습니다. 출처: 한국관광공사(data.go.kr). 라이선스: 공공누리 제3유형." — 앱 푸터에 상시 표기.
> **변경금지 준수**: 원본 값은 변형하지 않고 필요한 필드 부분집합만 선택하여 사용.

### 7.2 데이터 모델 (스키마 개요)

- **관광 데이터(읽기 전용)**: `manifest.json` + 카테고리별 `{region, contentType, total, items[]}`. item 주요 필드: `contentid, title, addr1, tel, mapx(경도), mapy(위도), firstimage, modifiedtime, lDongSignguCd(자치구)`.
- **게시글(localStorage `localhub.posts`)**: `{ id, region, title, content, password, author, tags[], createdAt, updatedAt, views, likes }`
- **부가 상태**: `localhub.bookmarks[]`, `localhub.liked[]`, `localhub.lastSeenPostId`

## 8. API 명세 (개요)

> 별도 백엔드 API 없음. 외부 호출은 아래 2종.

| Method | Endpoint | 설명 | 요청 | 응답 |
|--------|----------|------|------|------|
| GET | `/data/seoul/{category}.json` (정적) | 제공 관광 데이터 로드 | — | `{ items[] }` |

## 10. 성공 기준 (Definition of Done)

- 5개 권역 게시판에서 CRUD가 동작하고, 올바른 비밀번호로만 수정·삭제된다.
- 챗봇이 제공 JSON을 근거로 응답한다 (키 없으면 로컬 검색으로 폴백).
- 지도/대시보드/캘린더가 실제 서울 데이터로 렌더링된다.
- `npm run build` 성공 및 Netlify 배포 URL에서 정상 동작한다.
- `.env`(API 키)가 저장소에 포함되지 않는다.

## 11. 리스크 및 대응 (Risks)

| 리스크 | 영향 | 대응 방안 |
|--------|------|-----------|
| `VITE_` 키가 빌드 산출물에 노출 | 상 | 사용량·결제 한도가 낮은 키만 사용, `.env` 커밋 금지, 키 없을 시 로컬 검색 폴백 |
| 대용량 JSON(쇼핑 4천여 건) 렌더 성능 | 중 | 카테고리별 지연 로드 + 캐시, 지도 마커 상한(800) |
| localStorage 기기 간 미동기화 | 중 | UI에 "작성 브라우저에만 저장" 안내 명시 (의도된 한계) |
| 제공 데이터에 행사 일정 필드 부재 | 하 | 캘린더는 정보 갱신일 기준으로 대체, 문서·UI에 명시 |
