# LocalHub 개발 진행 순서

실제 개발이 진행된 순서를 단계별로 정리한 문서입니다. 각 단계 종료 시 로컬 git에 커밋을 남겼습니다.

## 0. 사전 분석
1. LocalHub 공문(v8) 정독 → 요구사항/제약 추출 (백엔드 없는 정적 SPA, 익명 CRUD, 챗봇, Netlify).
2. 제공 산출물 템플릿(MVP, WBS/Gantt) 구조 파악.
3. 제공 데이터(서울 zip) 스키마·라이선스(SCHEMA.md/SOURCE.md) 확인 → 공공누리 3유형, TourAPI 8종.
4. 개발 가능성 판단 및 계획 수립 → **사용자 확인**(챗봇 방식/선택기능 범위) 후 착수.

## 1. 프로젝트 스캐폴딩 & 데이터 준비  · `commit: chore`
1. `git init` (기존 폴더가 저장소가 아니어서 초기화).
2. Vite + Vue3 프로젝트 구성: `package.json`, `vite.config.js`, `index.html`, `.env.example`.
3. 서울 zip → `public/data/seoul/*.json` 정규화(파일명 영문화 + 필드 부분집합) 및 `manifest.json` 생성.
4. 서비스 계층(`dataLoader`, `storage`, `openai`), 상수(`regions`, `categories`), 전역 CSS 작성.
5. `npm install`.

## 2. 앱 셸 (라우터/레이아웃/홈)  · `commit: feat`
1. `router/index.js`(지연 로딩 라우트), `main.js`, `App.vue`.
2. `AppHeader`(반응형 네비), 데이터 출처 푸터.
3. `HomeView` — 5권역 카드, 카테고리 요약, 최근 게시글, `manifest` 로드.

## 3. 익명 커뮤니티 CRUD  · `commit: feat`
1. `stores/posts.js`(Pinia) — 생성/조회/수정/삭제 + localStorage 영속화 + 데모 시드.
2. `BoardListView`(권역 탭·검색·태그·페이지네이션), `PostDetailView`, `PostEditView`.
3. `PasswordModal` — 수정·삭제 시 비밀번호 프론트 검증.
4. 조회수·좋아요·북마크·공유 등 부가기능.

## 4. 지역정보 챗봇  · `commit: feat`
1. `services/openai.js` — 키가 있으면 OpenAI API RAG 호출, 없으면 로컬 키워드 검색 폴백.
2. `ChatWidget` — 플로팅/펼침, 대화 히스토리, 예시 질문, 모바일 전체화면, 근거 출처 표기.

## 5. 선택기능 5종  · `commit: feat`
1. `MapView` — Leaflet + OSM, 카테고리별 POI(circleMarker), 마커 상한.
2. `DashboardView` — Chart.js 도넛/막대 3종(카테고리·권역·자치구 TOP10).
3. `CalendarView` — 축제 월별 캘린더(정보 갱신일 기준).
4. `NewPostNotifier` — localStorage polling 새 글 토스트 알림.
5. 커뮤니티 부가기능(검색/태그/좋아요/북마크/공유)은 3단계에 통합.

## 6. 검증 & 문서화
1. `npm run build` 성공 확인(75 모듈, 오류 0).
2. 개발 서버 + 브라우저 프리뷰로 홈/지도/대시보드/챗봇/CRUD/알림 실동작 확인.
3. 산출물 문서 작성: MVP 템플릿·MVP 정의서·WBS/Gantt·README·진행순서·에러 로그.

## 7. 남은 단계 (제출 전)
1. `.env` 생성 후 `VITE_OPENAI_API_KEY` 입력 → 챗봇 AI 모드 확인 (선택).
2. Netlify에 repo 연동 배포 → 배포 URL 산출물에 기재.
3. 발표 PPT 작성.

---

### 실행 방법 요약
```bash
cd localhub-frontend
npm install
cp .env.example .env      # (선택) OpenAI API 키 입력
npm run dev               # http://localhost:5173
npm run build             # 프로덕션 빌드 → dist/
```
